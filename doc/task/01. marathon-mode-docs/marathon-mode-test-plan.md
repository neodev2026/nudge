# Test Plan: Marathon Mode

**작성일**: 2026-04-23
**최종 업데이트**: 2026-04-23 (v3)
**대상 기능**: Marathon Mode (NUDGE-042)
**테스트 환경**: Staging (nudge.neowithai.com) + 로컬 개발 서버

**사전 조건**:
- Deutsch A1 상품 구독 완료된 테스트 계정 1개 (account-A)
- 구독 없는 테스트 계정 1개 (account-B)
- 비로그인 브라우저 1개

---

## TC-MM-01: 진입점 표시

**분류**: UI
**사전 조건**: account-A로 로그인, `/sessions/:sessionId` 방문
**절차**:
1. session-choice-page에 접속한다.
2. 표시되는 옵션을 확인한다.

**기대 결과**: "학습 목록", "Leni와 학습", "마라톤 모드" 세 가지 옵션이 표시된다.

**사전 조건**: 비로그인 상태
**절차**:

**절차**:

**기대 결과**: 구독 유도 화면 또는 상품 상세 페이지로 리디렉트된다.

---

## TC-MM-04: 신규 Run 생성

**분류**: 기능 / DB
**사전 조건**: account-A, 기존 marathon run 없음
**절차**:
1. session-choice-page에서 "마라톤 모드 →"를 클릭하고 시작한다.
2. DB `nv2_marathon_runs`를 확인한다.

**기대 결과**:
- `status = 'in_progress'`, `run_number = 1`, `last_stage_index = 0`인 row가 생성된다.
- 화면은 첫 번째 스테이지의 첫 번째 카드부터 시작한다.

---

## TC-MM-05: 카드 표시 순서

**분류**: 기능
**사전 조건**: Marathon Mode 스트림 진행 중
**절차**:
1. 첫 5개 스테이지를 진행하며 표시되는 스테이지 순서를 기록한다.
2. DB의 `nv2_product_session_stages.display_order` 값과 비교한다.

**기대 결과**: 화면 표시 순서가 `display_order` ASC 순서와 일치한다.

---

## TC-MM-06: 스테이지 내 카드 전부 표시

**분류**: 기능
**사전 조건**: Marathon Mode 스트림 진행 중
**절차**:
1. 첫 번째 스테이지 진행 중 표시되는 카드 수를 센다.
2. DB에서 해당 stage_id에 연결된 카드 수를 확인한다.

**기대 결과**: 화면에 표시된 카드 수와 DB의 카드 수가 일치한다.

---

## TC-MM-07: TTS 자동 재생

**분류**: 기능
**사전 조건**: Marathon Mode 스트림 진행 중
**절차**:
1. 발음 듣기 버튼이 있는 카드(단어/예문) 진입 시 소리를 확인한다.
2. 설명 카드(발음 버튼 없음) 진입 시 소리를 확인한다.

**기대 결과**:
- 발음 버튼 있는 카드: TTS 2회 자동 재생된다.
- 발음 버튼 없는 카드: TTS 재생 없음.

---

## TC-MM-08: 미니 퀴즈 출제

**분류**: 기능
**사전 조건**: Marathon Mode 스트림 진행 중
**절차**:
1. 카드를 넘겨 5개 스테이지를 완료한다.
2. 미니 퀴즈 화면을 확인한다.

**기대 결과**:
- 5문항이 표시된다.
- 각 문항은 방금 본 5개 스테이지 중 하나를 대상으로 한다.
- `word_to_meaning`과 `meaning_to_word` 유형이 혼합되어 있다.
- 오답 보기 3개는 모두 같은 상품 내 다른 스테이지에서 추출된다.

---

## TC-MM-09: 복습 퀴즈 대체 출제

**분류**: 기능
**사전 조건**: Marathon Mode 스트림 진행 중
**절차**:
1. 50번째 스테이지까지 진행한다.
2. 퀴즈 화면을 확인한다.

**기대 결과**:
- 미니 퀴즈(5문항) 대신 복습 퀴즈(50문항)가 표시된다.
- 문항 제목 또는 헤더에 "복습 퀴즈"임을 구분하는 표시가 있다.

---

## TC-MM-10: 복습 퀴즈 범위 — 직전 50개 (기본)

**분류**: 기능
**사전 조건**: "이전 내용 포함" 설정 OFF, 100번째 스테이지 완료
**절차**:
1. 복습 퀴즈 문항 수를 확인한다.
2. 출제 문항의 단어가 51~100번째 스테이지 범위 내인지 확인한다.

**기대 결과**: 50문항이 출제되며, 51~100번째 스테이지 단어만 출제된다.

---

## TC-MM-11: 복습 퀴즈 범위 — 누적 전체 (설정 ON)

**분류**: 기능
**사전 조건**: "이전 내용 포함" 설정 ON, 100번째 스테이지 완료
**절차**:
1. 복습 퀴즈 문항 수를 확인한다.
2. 출제 문항의 단어 범위를 확인한다.

**기대 결과**: 100문항이 출제되며, 1~100번째 스테이지 단어 전체가 출제 범위에 포함된다.

## TC-MM-13: 이어하기 (Resume)

**분류**: 기능
**사전 조건**: Marathon Mode 진행 중 일시정지
**절차**:
1. 12번째 스테이지 진행 중 일시정지 버튼을 누른다.
2. 브라우저를 닫는다.
3. 다시 session-choice-page에서 "마라톤 모드 →"를 클릭한다.

**기대 결과**:
- "N번째 단어부터 이어하기"와 "처음부터" 버튼이 표시된다.
- "이어하기" 클릭 시 13번째 스테이지 첫 카드부터 재개된다.

## TC-MM-14: 처음부터 재시작

**분류**: 기능
**사전 조건**: 기존 `in_progress` run이 있는 상태
**절차**:
1. Marathon Mode 진입 후 "처음부터" 버튼을 클릭한다.
2. DB `nv2_marathon_runs`를 확인한다.

**기대 결과**:
- 새 run이 생성되거나 기존 run이 초기화된다.
- 화면은 첫 번째 스테이지 첫 카드부터 시작한다.

---

## TC-MM-15: 전체 퀴즈 완료 및 완주 기록 저장

**분류**: 기능 / DB
**사전 조건**: 전체 스테이지 스트림 완료 후 전체 퀴즈 진행
**절차**:
1. 전체 퀴즈 마지막 문항에 답변한다.
2. DB `nv2_marathon_runs`와 `nv2_marathon_answers`를 확인한다.

**기대 결과**:
- `nv2_marathon_runs`: `status = 'completed'`, `score`, `total_questions`, `completed_at`, `elapsed_seconds` 저장됨.
- `nv2_marathon_answers`: 전체 스테이지 수만큼 row 생성, `is_correct` 값이 실제 정오와 일치.
- 완주 결과 화면이 표시된다.

---

## TC-MM-16: 완주 추이 테이블

**분류**: UI / 데이터
**사전 조건**: 2회 이상 완주 완료
**절차**:
1. 두 번째 완주를 완료한다.
2. 완주 결과 화면의 추이 테이블을 확인한다.

**기대 결과**:
- 회차별 점수(N/전체)와 소요 시간이 나란히 표시되는 테이블이 표시된다.
- 1회차와 2회차 결과가 비교 가능하다.
- 스테이지별 정오 세부 테이블은 표시되지 않는다.
