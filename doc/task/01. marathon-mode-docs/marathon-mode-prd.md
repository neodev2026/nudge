## 1. 배경 및 목적

Nudge v2의 기존 학습 흐름은 세션(5개 단어) 단위로 DM 알림을 받아 학습하는 구조다.
이 구조는 꾸준한 습관 형성에는 적합하지만, 특정 상품의 전체 어휘를 단기간에 집중 학습하려는 사용자 니즈를 충족하지 못한다.

Marathon Mode는 상품 내 전체 학습 스테이지를 중단 없이 한 번에 완주할 수 있는 집중 학습 도구다.
첫 번째 사용 사례는 Deutsch A1 전체 어휘(약 650개 단어 = 스테이지)를 반복 완주하며 암기하는 것이다.

---

## 2. 목표

- 사용자가 상품 전체 스테이지를 1회 완주하는 경험을 제공한다.
- 완주를 반복할수록 전체 퀴즈에서 맞힌 수가 늘어나는 추이를 숫자로 확인할 수 있어야 한다.
- 중단 후 재접속 시 중단 위치에서 바로 이어서 학습할 수 있어야 한다.

---

## 3. 사용자 스토리

- 나는 Deutsch A1 상품의 Marathon Mode를 시작하여 전체 단어를 스트림으로 빠르게 보고 미니 퀴즈로 즉시 확인한 뒤 완주 기록을 남기고 싶다.
- 나는 오늘 완주에서 100개를 맞혔고, 내일 다시 완주해서 200개를 맞히는 성장 추이를 보고 싶다.
- 나는 학습 중 잠시 멈추고 나중에 돌아왔을 때 중단된 위치에서 바로 이어서 학습하고 싶다.
- 나는 전체 퀴즈에서 어떤 단어를 매번 틀리는지 회차별 상세 기록으로 확인하고 싶다.
- 나는 복습 퀴즈 범위를 직전 50개로 할지 처음부터 누적으로 할지 설정하고 싶다.

## 4. 기능 범위 (Scope)

### 4-1. In Scope (이번 구현)

**A. Marathon 스트림 (학습 화면)**
- 상품 내 모든 learning stage의 카드를 `display_order` 순서대로 연속 표시
- 1개 스테이지 = 해당 스테이지에 등록된 카드 전부를 순서대로 표시 (카드 수에 무관)
- 기존 stage-page UI 재활용
- 카드 진입 시 TTS 자동 2회 재생 (발음 듣기 버튼이 있는 카드에 한함)
- 다음 버튼: 즉시 수동 넘김
- 자동 넘김 옵션: 다음 버튼 옆 카운트다운 표시 → TTS 완료 후 타이머 시작
- 일시정지 버튼: 즉시 위치(스테이지 단위) 저장 후 중단
- 진행률 바: 전체 스테이지 수 대비 현재 스테이지 순번 표시

**B. 미니 퀴즈 (5개 스테이지마다)**
- 방금 본 5개 스테이지(단어) 전체를 대상으로 5문항 출제
- 50번째 스테이지 도달 시 미니 퀴즈 대신 복습 퀴즈 실행
- 문항 유형: `단어 → 의미 고르기`와 `의미 → 단어 고르기` 혼합
- 오답 보기: 같은 상품 내 다른 스테이지에서 무작위 3개 추출
- 시간제한: 기본 OFF / 사용자 설정으로 ON 가능 (5 · 8 · 10 · 15초)
- 결과: 문항별 정오 즉시 표시 → 확인 버튼으로 다음 스트림 진행
- **결과 미저장** (화면 표시만)

**C. 복습 퀴즈 (50개 스테이지마다)**
- 미니 퀴즈와 동일한 형식 (혼합 4지선다)
- 문항 수: 사용자 설정에 따라 결정
  - 기본 (체크 OFF): 50문항 (직전 50개 스테이지 대상)
  - 누적 (체크 ON): 처음부터 현재까지 스테이지 수만큼 (예: 100번째 도달 시 100문항)
- 출제 범위: 사용자 설정에 따라 선택
  - 기본 (체크 OFF): 직전 50개 스테이지
  - 누적 (체크 ON): 처음부터 현재까지 전체 스테이지
- 시간제한: 미니 퀴즈와 동일한 설정 공유
- 결과: 문항별 정오 즉시 표시 → 확인 버튼으로 다음 스트림 진행
- **결과 미저장** (화면 표시만)

**D. 전체 퀴즈 (Final Quiz)**
- 전체 스테이지 완료 후 상품 내 전체 스테이지 대상 퀴즈 실시
- 미니/복습 퀴즈와 동일한 형식 (혼합 4지선다)
- 문항 수: 전체 스테이지 수와 동일
- 다 못 맞혀도 통과 — 완주 기록 저장 후 결과 화면 표시
- **결과 저장**: 스테이지별 정오를 `nv2_marathon_answers`에 저장

**E. 완주 기록 (Run Record)**
- 저장 항목: 상품 ID, 완주 회차, 완주 날짜, 전체 퀴즈 점수(N/전체), 소요 시간
- 세부 항목: 스테이지별 정오 — 전체 퀴즈만 저장 (오답 모드 등 향후 활용 목적)
- 결과 화면: 이번 완주 점수 + 회차별 점수/소요 시간 추이 테이블 (스테이지별 정오 테이블 미표시)

**F. 진입점 및 이어하기**
- 진입: `session-choice-page` 세 번째 옵션 "마라톤 모드 →"
- 이어하기: 미완주 기록이 있으면 "N번째 단어부터 이어하기" 버튼 표시
- 처음부터: 항상 선택 가능
  - 기존 in_progress run이 있으면: UPDATE (last_stage_index=0, started_at=now, score/elapsed_seconds/completed_at=null)
  - 기존 completed run만 있으면: 신규 run INSERT (run_number = completed count + 1)
  - run 없으면: 신규 run INSERT (run_number = 1)

**G. 출력 (Print)**
- 별도 라우트 `/products/:slug/marathon/print` 추가
- Marathon Mode 진입 화면에서 "전체 출력" 버튼 → 새 탭으로 열기
- A4 시트: 단어 / 의미 / 예문(빈칸) / 쓰기 3회 구성 (기존 포맷 동일)

### 4-2. Out of Scope (이번 미구현, 추후 고려)

- 틀린 단어만 재시도하는 "오답 모드" (완주 데이터 확보 후 2차 기획)
- 상품 간 혼합 학습 (A1+A2 동시)
- 리더보드 / 다른 사용자와의 비교
- 모바일 앱 네이티브 지원

## 5. UI 흐름

```
session-choice-page
└─ [마라톤 모드 →] 클릭
   └─ /products/:slug/marathon
      ├─ 미완주 기록 있음 → "N번째 단어부터 이어하기" / "처음부터" 선택
      └─ 미완주 없음 → 바로 스트림 시작

스트림 루프:
  스테이지 진입 → 해당 스테이지의 카드 전부 순서대로 표시 (TTS 자동 재생)
  → 스테이지 내 마지막 카드 → 다음 스테이지로 (위치 자동 저장)
  → 5개 스테이지 완료 → 미니 퀴즈 5문항 (단, 50번째 도달 시 복습 퀴즈로 대체)
  → 결과 표시 (미저장) → 다음 5개 스테이지
  → 50개 스테이지 완료 → 복습 퀴즈 50문항
  → 결과 표시 (미저장) → 다음 스테이지
  → ...반복...
  → 전체 스테이지 완료 → 전체 퀴즈 (전체 스테이지 수만큼 문항)
  → 완주 결과 화면 (점수 + 추이 테이블) ← DB 저장
  → "다시 완주" or "종료"
```

## 6. 데이터 모델 (신규 테이블 2개)

### nv2_marathon_runs

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid PK | |
| auth_user_id | text | FK → auth.users |
| product_id | uuid | FK → nv2_learning_products |
| run_number | integer | 이 상품의 몇 번째 완주인지 |
| status | text | `in_progress` / `completed` |
| score | integer | 전체 퀴즈 정답 수 (완주 시 저장) |
| total_questions | integer | 전체 퀴즈 문항 수 (완주 시 저장) |
| last_stage_index | integer | 중단 위치 스테이지 순번 (이어하기용, 0-based) |
| elapsed_seconds | integer | 완주 소요 시간 |
| started_at | timestamptz | |
| completed_at | timestamptz | nullable |

### nv2_marathon_answers

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid PK | |
| run_id | uuid | FK → nv2_marathon_runs |
| stage_id | uuid | FK → nv2_stages (단어 단위 식별) |
| question_direction | text | `word_to_meaning` / `meaning_to_word` |
| is_correct | boolean | |
| answered_at | timestamptz | |

> 미니/복습 퀴즈 결과는 저장하지 않으므로 `quiz_type` 컬럼 불필요.
> 퀴즈 문항은 스테이지(단어) 단위로 출제되므로 `card_id` 불필요, `stage_id`로 충분.

## 7. Marathon 설정 항목

| 설정 항목 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| 자동 넘김 | toggle | OFF | ON 시 TTS 완료 후 카운트다운 → 자동 다음 카드 |
| 자동 넘김 대기 시간 | select | 3초 | 1 · 2 · 3 · 5초 선택 |
| 퀴즈 시간제한 | toggle | OFF | 미니/복습/전체 퀴즈 공통 적용 |
| 퀴즈 시간제한 시간 | select | 8초 | 5 · 8 · 10 · 15초 선택 |
| 복습 퀴즈 누적 포함 | checkbox | OFF | ON 시 복습 퀴즈 범위 = 처음부터 현재까지 누적 |

> 설정 저장 위치: `localStorage` (구현 단순화) 또는 `nv2_profiles` 컬럼 확장 (구현 시 결정)

## 8. Acceptance Criteria

| # | 조건 | 기준 |
|---|---|---|
| AC-01 | 스트림 시작 | session-choice-page에서 "마라톤 모드" 선택 시 `/products/:slug/marathon`으로 이동한다 |
| AC-02 | 카드 연속 표시 | 모든 learning stage의 카드가 `display_order` 순서대로 스테이지 단위로 끊김 없이 표시된다 |
| AC-03 | TTS 자동 재생 | 발음 듣기 버튼이 있는 카드 진입 시 TTS가 자동으로 2회 재생된다 |
| AC-04 | 미니 퀴즈 | 5개 스테이지마다 미니 퀴즈 5문항이 출제된다 (방금 본 5개 스테이지 대상) |
| AC-05 | 복습 퀴즈 대체 | 50번째 스테이지 완료 시 미니 퀴즈 대신 복습 퀴즈 50문항이 출제된다 |
| AC-06 | 복습 퀴즈 범위 설정 | "이전 내용 포함" 체크 ON 시 복습 퀴즈가 처음부터 현재까지 누적 범위로 출제된다 |
| AC-07 | 퀴즈 방향 혼합 | 미니/복습/전체 퀴즈 모두 `word_to_meaning`과 `meaning_to_word`가 혼합 출제된다 |
| AC-08 | 미니/복습 퀴즈 미저장 | 미니 퀴즈 및 복습 퀴즈 결과는 DB에 저장되지 않는다 |
| AC-09 | 이어하기 | 중단 후 재접속 시 `last_stage_index` 위치의 스테이지부터 스트림이 재개된다 |
| AC-10 | 완주 기록 저장 | 전체 퀴즈 완료 시 `nv2_marathon_runs`에 `completed` 상태로 저장된다 |
| AC-11 | 세부 기록 저장 | 전체 퀴즈의 모든 문항 정오가 `nv2_marathon_answers`에 `stage_id` 기준으로 저장된다 |
| AC-12 | 추이 테이블 | 완주 결과 화면에서 회차별 점수(N/전체)와 소요 시간 추이 테이블이 표시된다 |
| AC-13 | 전체 출력 | "전체 출력" 버튼 클릭 시 `/products/:slug/marathon/print`가 새 탭으로 열리며 상품 전체 카드가 A4 프린트 시트로 렌더링된다 |

## 9. 비기능 요건

- 로그인 필수 (익명 사용자는 마라톤 모드 진입 불가, 로그인 유도)
- 상품 구독 상태 확인 후 진입 허용
- 카드 순서는 `nv2_product_session_stages.display_order` 기준 고정 (랜덤 없음)
- 미니/복습/전체 퀴즈 오답 보기는 같은 `product_id` 내 스테이지에서 추출 (타 상품 혼입 금지)
- 전체 카드를 marathon-page loader에서 한 번에 로딩하여 클라이언트 상태로 관리
  (네트워크 지연 없이 즉각적인 카드 전환을 위해)
